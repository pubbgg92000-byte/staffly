# Staffly Release Notes — v0.23.2

**Tag:** `v0.23.2-deploy` · **Branch:** `feat/v0.23.2-deploy` · **Date:** 2026-06-09
**Theme:** Deployment hardening, demo readiness, and pre-production certification.

This release moves Staffly from feature-complete-on-localhost toward a
publicly hostable demo / pilot. It adds the production-deployment surface
(reverse proxy, process manager, tunnel, object storage, monitoring, CI),
a rich deterministic demo dataset, and a battery of robustness fixes found
during a pre-production certification audit.

---

## New Features

- **Demo-readiness seed** (`pnpm --filter @staffly/api db:seed:demo`):
  deterministic, idempotent "Acme Corporation" tenant — 40 employees, manager
  hierarchy, 6 locations / 8 departments / 13 designations, 90 days of
  attendance (including in-progress today), 5 leave types, 160 balances, 54
  leave requests in every state, 6 announcements, 24 documents, 24
  notifications. Re-running converges to the same state and touches only the
  `staffly-demo` org.
- **Manager role** gains team reads + leave approval (see Known Issues for the
  scoping caveat).
- **Error boundaries** in both portals; **Forbidden** empty-state on 403.

## Improvements

- **Manager team-scoping (row-level)** — managers now see/act on only their
  direct + indirect reports (verified live: 10 of 40 employees); hr/super keep
  org-wide access.
- **Session-expiry handling** — a 401 from any data query clears state, toasts
  "Session expired", and redirects to sign-in (`logout` is `@Public` to clear
  cookies on an expired token).
- Admin dashboard refreshes instantly after employee and attendance mutations.
- Documentation suite: README demo section, `DEMO_GUIDE.md`, `TESTING.md`,
  `RUNBOOK.md`, `DEPLOYMENT_READINESS.md`, `DEPLOYMENT.md`, this file.

## Security Enhancements

- Helmet headers; throttler (global 120/min, auth 10/min, CF-IP keyed).
- Real `/readyz` dependency probe (PostgreSQL + storage) → 503 on failure.
- Verified live: invalid login / tampered JWT / missing cookie → 401;
  privilege escalation → 403; CSRF-less mutation → 403; parameterized queries
  (SQLi attempt returns no rows, table intact).

## Infrastructure Changes

- Cloudflare R2 storage client (AWS SDK v3); Sentry on API + portals.
- CI: unit + integration + artifact jobs.
- Deployment assets: prod compose (PG18), PM2, Caddy, Cloudflare Tunnel,
  `deploy/*.sh`.

## Bug Fixes

- Dashboard staleness after mutations (employee + attendance) resolved.
- "Today's attendance" and "New joins this month" widgets populated by the seed.
- v0.23.1 timezone fix carried forward.

## Known Issues

See [CHANGELOG.md](../../CHANGELOG.md#v0232--2026-06-09). Headlines: manager
cannot reject leave (`leave.reject` not granted); email delivery is not wired;
announcement `bodyHtml` is rendered as HTML for privileged authors. (Manager
team-scoping is now row-enforced and session-expiry now redirects with a
toast — resolved in this release.)

## Upgrade Notes

1. `prisma migrate deploy`
2. `pnpm --filter @staffly/api db:seed` then `db:seed:demo`
3. Re-login any open browser sessions (reseed rotates user IDs).
4. Set production env: `COOKIE_DOMAIN`, `CORS_ORIGINS`, R2 `S3_*`, optional
   `SENTRY_DSN`.
